## 二、發現的問題

### 🔴 高優先級（必須修復）

#### 1. Tours API 測試失敗（5 個測試）
**問題**：`tours.create` API 缺少必填欄位 `destinationCountry` 和 `destinationCity`
**原因**：測試檔案未更新以包含新增的必填欄位
**影響**：無法正確測試行程建立、更新、刪除功能
**修復方案**：更新 `server/tours.test.ts` 中的測試資料，加入 `destinationCountry` 和 `destinationCity` 欄位

#### 2. Auth Logout 測試失敗（1 個測試）
**問題**：Cookie 選項不匹配，預期 `maxAge: -1` 但實際未設定
**原因**：登出時 cookie 清除邏輯與測試預期不一致
**影響**：登出功能可能無法正確清除 session
**修復方案**：更新 `server/routers.ts` 中的 logout 邏輯或更新測試預期

#### 3. Google OAuth 登入後 Header 不顯示用戶資訊
**問題**：登入成功但 Header 仍顯示「登入 / 註冊」
**原因**：Cookie 設定或前端認證狀態讀取問題
**影響**：用戶體驗不佳，無法確認登入狀態
**修復方案**：檢查 cookie 設定和前端 useAuth hook

### 🟡 中優先級（建議修復）

#### 4. 開發環境端口衝突
**問題**：端口 3000 經常被殭屍進程佔用
**影響**：開發體驗不佳，需要頻繁重啟
**修復方案**：已在程式碼中加入端口重用邏輯（SO_REUSEADDR）

#### 5. 缺少部分功能頁面
**問題**：代辦簽證、包團旅遊、機票預購、機場接送、飯店預訂頁面內容不完整
**影響**：用戶無法使用這些服務
**修復方案**：逐步完善各服務頁面

### 🟢 低優先級（可延後）

#### 6. baseline-browser-mapping 套件過期
**問題**：套件資料超過兩個月未更新
**影響**：可能影響瀏覽器相容性檢測準確度
**修復方案**：執行 `npm i baseline-browser-mapping@latest -D`
