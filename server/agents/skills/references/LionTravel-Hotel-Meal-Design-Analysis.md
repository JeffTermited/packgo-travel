## 一、飯店介紹設計

### 1.1 整體結構

**標題區塊:**
- 主標題: "保證入住"
- 副標題: "海上水上屋"
- 飯店名稱: "黃金棕櫚樹海上渡假村" (大字體,粗體)

**內容區塊:**
1. **飯店描述段落** (文字為主)
   - 詳細介紹飯店的位置、特色、房間數量
   - 設計理念和風格描述
   - 使用具體數字增加可信度 (例如: 393間海上渡假屋)

2. **注意事項** (分點列出)
   - 註1: 房型說明
   - 註2: 加床政策

3. **活動樂翻天** (分類列出)
   - ＊＊免費活動＊＊
     - 游泳池
     - 夜間活動 (3項)
   - ＊＊付費活動＊＊
     - 陸上運動 (5項)
     - 水上運動 (2項)
     - 健身課程 (1項)
   - 免責聲明

4. **圖片展示區** (3張圖片)
   - 飯店外觀
   - 飯店房型
   - 飯店活動

### 1.2 視覺設計特點

**排版:**
- 文字在上,圖片在下
- 圖片採用橫向排列 (3張並排)
- 每張圖片下方有簡短標題

**色彩:**
- 背景: 白色
- 文字: 黑色為主
- 標題: 使用較大字體和粗體

**間距:**
- 段落之間有明顯間距
- 圖片之間有適當間距

## 二、餐飲介紹設計

### 2.1 整體結構

**卡片式設計** (每道餐點一張卡片):

**卡片 1: 古宅娘惹風味**
- 小標題: "視覺味覺的雙重享受"
- 主標題: "古宅娘惹風味"

- 圖片: 餐點或餐廳環境照片

**卡片 2: 必比登推薦~海腳人海鮮粉**
- 小標題: "海味的噴發"
- 主標題: "必比登推薦~海腳人海鮮粉"
- 描述: "奶香湯底搭配著魚的鮮味，每一口都能感受到海鮮的甜味，很難令人不愛。"
- 圖片: 餐點照片

**卡片 3: 六宮格下午茶**
- 小標題: "邂逅南洋風情"
- 主標題: "六宮格下午茶"
- 描述: "將各式鹹甜點心精巧地擺放於六個獨立的宮格之中，呈現出視覺上的工整與豐富，享受一個充滿南洋韻味的寧靜午後。"
- 圖片: 下午茶照片

### 2.2 視覺設計特點

**排版:**
- 垂直堆疊卡片
- 每張卡片包含: 小標題 + 主標題 + 描述 + 圖片
- 圖片位於卡片底部

**色彩:**
- 背景: 白色或淺色
- 文字: 黑色為主
- 小標題: 可能使用品牌色或強調色

**字體:**
- 小標題: 較小字體
- 主標題: 大字體,粗體
- 描述: 中等字體

**圖片:**
- 高品質餐點照片
- 圖片尺寸一致
- 圖片比例約 16:9 或 4:3

## 三、設計原則總結

### 3.1 飯店介紹

1. **資訊完整性**
   - 提供詳細的飯店描述
   - 列出所有可用設施和活動
   - 明確標示免費和付費項目
   - 提供注意事項和政策說明

2. **視覺層次**
   - 文字內容在前,圖片在後
   - 使用標題、副標題、正文的層次結構
   - 圖片作為補充說明

3. **可讀性**
   - 使用分點列出活動項目
   - 段落清晰,間距適當
   - 重要資訊使用粗體或特殊符號標示

### 3.2 餐飲介紹

1. **情感化描述**
   - 使用感官詞彙 (視覺、味覺、嗅覺)
   - 強調文化體驗和情感連結
   - 描述具體的味道和口感

2. **品牌認證**
   - 提及"必比登推薦"等認證
   - 增加餐點的可信度和吸引力

3. **視覺吸引力**
   - 高品質餐點照片
   - 卡片式設計易於瀏覽
   - 每道餐點獨立呈現

## 四、實作建議

### 4.1 HotelAgent 輸出格式

```typescript
{
  title: string;           // 飯店名稱
  subtitle?: string;       // 副標題 (例如: "海上水上屋")
  badge?: string;          // 保證標籤 (例如: "保證入住")
  description: string;     // 詳細描述 (段落文字)
  roomTypes: string[];     // 房型說明
  policies: string[];      // 政策和注意事項
  facilities: {
    free: string[];        // 免費設施/活動
    paid: string[];        // 付費設施/活動
  };
  images: Array<{
    url: string;
    caption: string;       // 圖片說明 (例如: "飯店外觀")
  }>;
}
```

### 4.2 MealAgent 輸出格式

```typescript
{
  meals: Array<{
    subtitle: string;      // 小標題 (例如: "視覺味覺的雙重享受")
    name: string;          // 餐點名稱
    description: string;   // 情感化描述
    badge?: string;        // 認證標籤 (例如: "必比登推薦")
    image: string;         // 餐點圖片 URL
  }>
}
```

### 4.3 LionHotelSection 組件設計

- 使用單欄佈局
- 文字內容在上,圖片在下
- 圖片使用橫向網格排列 (3欄)
- 設施列表使用分類和分點顯示
- 注意事項使用編號列表

### 4.4 LionMealSection 組件設計

- 使用垂直堆疊的卡片佈局
- 每張卡片包含: 小標題 + 主標題 + 描述 + 圖片
- 圖片位於卡片底部
- 卡片之間有適當間距
- 支援 Hover 效果 (可選)
