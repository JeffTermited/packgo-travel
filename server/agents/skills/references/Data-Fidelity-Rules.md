# 資料忠實度規則

## 核心原則

> **「寧可留白，不可造假」**

當原始資料不足時，應該選擇標記為「待補充」，而非自行創造內容。

---

## 關鍵資訊檢查清單

### 1. 交通方式（最高優先級）⚠️

- [ ] 火車行程不能變成飛機行程
- [ ] 郵輪行程不能變成陸地行程
- [ ] 自駕行程不能變成團體巴士行程

**識別方法**：
- 標題或內容包含「鳴日號」「觀光列車」「火車」→ 交通方式 = 火車
- 標題或內容包含「郵輪」「遊輪」「cruise」→ 交通方式 = 郵輪
- 標題或內容包含「自駕」「租車」→ 交通方式 = 自駕

**致命錯誤範例**：
- ❌ 原始資料：「鳴日號火車行程」→ 生成結果：「飛機」
- ❌ 原始資料：「郵輪環遊」→ 生成結果：「巴士」

### 2. 住宿資訊

- [ ] 飯店名稱必須與原始資料完全一致
- [ ] 飯店等級不能隨意升降
- [ ] 不得將 A 飯店改為 B 飯店

**常見錯誤範例**：
- ❌ 「The GAYA Hotel 潮渡假酒店」改為「娜路彎大酒店」
- ❌ 「花蓮潔西艾美渡假酒店」改為「理想大地」
- ❌ 「五星級飯店」改為「民宿」

### 3. 景點資訊

- [ ] 景點名稱必須與原始資料一致
- [ ] 不得添加原始資料中沒有的景點
- [ ] 可以潤飾景點描述，但不能改變景點本身

**常見錯誤範例**：
- ❌ 原始資料：「普悠瑪部落」→ 生成結果：「鹿野高台」
- ❌ 添加原始資料中沒有的「三仙台日出」
- ❌ 將「成功海光走讀」改為「七星潭」

### 4. 餐食資訊

- [ ] 餐廳名稱必須與原始資料一致
- [ ] 不得隨意指定未提及的餐廳
- [ ] 餐食類型（早/午/晚餐）必須正確

---

## 處理策略

| 情況 | ✅ 正確做法 | ❌ 錯誤做法 |
|------|-----------|-----------|
| 缺少景點描述 | 標記為「待補充」或使用簡短描述 | 自行創造詳細描述 |
| 缺少時間點 | 使用「上午」「下午」等模糊描述 | 隨意指定具體時間如「09:30」 |
| 缺少飯店名稱 | 使用「精選飯店」或「當地飯店」 | 隨意指定具體飯店名稱 |
| 缺少餐食資訊 | 使用「當地特色餐」或「自理」 | 隨意指定具體餐廳 |
| 缺少交通方式 | 使用「專車接送」或「待確認」 | 隨意指定飛機或火車 |

---

## 允許的行為 ✅

### 1. 潤飾描述
可以美化景點描述的文字，但不能改變事實。

**範例**：
- ✅ 「太魯閣」→ 「太魯閣國家公園：大自然的鬼斧神工」
- ✅ 「普悠瑪部落」→ 「普悠瑪部落：體驗卑南族傳統文化」

### 2. 添加合理時間
可以根據邏輯推斷添加時間點。

**範例**：
- ✅ 添加「09:00 出發」（如果原始資料提到上午出發）
- ✅ 添加「12:00 午餐」（合理的用餐時間）

### 3. 補充背景知識
可以補充景點的背景資訊。

**範例**：
- ✅ 補充「太魯閣國家公園成立於1986年」
- ✅ 補充「鳴日號是台鐵觀光列車」

---

## 禁止的行為 ❌

### 1. 更改交通方式
絕對禁止更改原始資料中的交通方式。

**範例**：
- ❌ 「鳴日號火車」改為「飛機」
- ❌ 「郵輪」改為「巴士」
- ❌ 「自駕」改為「團體遊覽車」

### 2. 更改飯店名稱
絕對禁止更改原始資料中的飯店名稱。

**範例**：
- ❌ 「The GAYA Hotel」改為「娜路彎大酒店」
- ❌ 「潔西艾美」改為「理想大地」

### 3. 添加新景點
絕對禁止添加原始資料中沒有的景點。

**範例**：
- ❌ 添加原始資料中沒有的「鹿野高台」
- ❌ 添加原始資料中沒有的「三仙台」
- ❌ 添加原始資料中沒有的「七星潭」

---

## 忠實度評分標準

| 分數 | 標準 |
|------|------|
| 100 | 完全忠於原始資料，無任何創造 |
| 80-99 | 主要資訊正確，僅有輕微潤飾 |
| 60-79 | 部分資訊不一致，需要修復 |
| 40-59 | 多處資訊錯誤，需要重新生成 |
| 0-39 | 嚴重偏離原始資料，不可接受 |

**致命錯誤（直接 0 分）**：
- 交通方式錯誤（火車→飛機）
- 飯店名稱完全錯誤
- 添加大量不存在的景點

---

## 驗證檢查清單

在生成完成後，必須檢查：

```
✅ 交通方式檢查
- [ ] 與原始資料一致
- [ ] 沒有將火車改為飛機
- [ ] 沒有將郵輪改為陸地交通

✅ 飯店名稱檢查
- [ ] 每天的飯店名稱與原始資料一致
- [ ] 沒有替換為其他飯店

✅ 景點檢查
- [ ] 所有景點都來自原始資料
- [ ] 沒有添加新景點
- [ ] 景點順序合理

✅ 餐食檢查
- [ ] 餐廳名稱與原始資料一致（如有提及）
- [ ] 餐食類型正確
```
