# PACK&GO AI 自動生成行程系統 - 架構流程說明

## 📅 今日完成項目（2026-01-25）

### ✅ 階段一：基礎架構（100% 完成）
1. **Redis + BullMQ 任務佇列系統**
   - Redis 連線配置（server/redis.ts）
   - BullMQ 任務佇列（server/queue.ts）
   - Worker 並發處理（2 個任務同時執行）
   - 速率限制（行程生成 5/小時、圖片生成 20/小時）

2. **資料庫擴充**
   - 新增 6 個欄位：heroImage, heroImageAlt, heroSubtitle, colorTheme, keyFeatures, poeticContent
   - 新增 5 個欄位：poeticSubtitle, attractions, hotels, meals, flights
   - 新增 2 個欄位：sourceUrl, originalityScore

3. **tRPC API**
   - `generateFromUrl`：生成行程（async）
   - `getGenerationStatus`：查詢任務狀態
   - `getMyGenerationJobs`：查詢用戶所有任務

4. **速率限制機制**
   - 使用 Redis sliding window 演算法
   - 行程生成：5 次/小時
   - 圖片生成：20 次/小時

### ✅ 階段二：多代理系統（100% 完成）
1. **Web Scraper Agent**（server/agents/webScraperAgent.ts）
   - 從旅遊網站抓取原始資料
   - 資料驗證機制

2. **Content Analyzer Agent**（server/agents/contentAnalyzerAgent.ts）
   - 標題改寫（行銷導向）
   - 描述改寫（100-150 字）
   - Hero 副標題生成
   - 亮點生成（3-5 個）
   - 關鍵特色生成（直式排版）
   - 詩意文案生成
   - 原創性驗證（60-100 分）

3. **Image Prompt Agent**（server/agents/imagePromptAgent.ts）
   - LLM 優化提示詞
   - Hero 圖片提示詞（100-150 字）
   - 亮點圖片提示詞（80-100 字）
   - 特色圖片提示詞（80-100 字）

4. **Image Generation Agent**（server/agents/imageGenerationAgent.ts）
   - Manus API 生成 Hero 圖片

   - 自動上傳至 S3

5. **Color Theme Agent**（server/agents/colorThemeAgent.ts）

6. **Master Agent**（server/agents/masterAgent.ts）
   - 協調所有 Agent 執行
   - 進度追蹤（7 個步驟）
   - 錯誤處理和回滾機制

## 🏗️ 系統架構流程

### 1. 用戶觸發生成
```
用戶輸入旅遊網站 URL
  ↓
tRPC API: generateFromUrl
  ↓
生成唯一 requestId
  ↓
將任務加入 BullMQ 佇列
  ↓
返回 jobId 給前端
```

### 2. Worker 處理任務
```
BullMQ Worker 接收任務
  ↓
Master Agent 開始協調
  ↓
執行多代理系統（7 個步驟）
  ↓
儲存至資料庫
  ↓
更新任務狀態為 completed
```

### 3. 多代理系統流程
```
Step 1: Web Scraper Agent（10%）
  ├─ 抓取網頁內容
  ├─ 使用 LLM 提取結構化資料
  └─ 驗證資料完整性
  
Step 2: Content Analyzer Agent（30%）
  ├─ 標題改寫（行銷導向）
  ├─ 描述改寫（100-150 字）
  ├─ Hero 副標題生成
  ├─ 亮點生成（3-5 個）
  ├─ 關鍵特色生成（直式排版）
  ├─ 詩意文案生成
  └─ 原創性驗證（60-100 分）
  
Step 3: Color Theme Agent（40%）
  ├─ 根據目的地生成配色主題
  └─ 返回 7 個顏色變數
  
Step 4: Image Prompt Agent（50%）
  ├─ 生成 Hero 圖片提示詞
  ├─ 生成亮點圖片提示詞
  └─ 生成特色圖片提示詞
  
Step 5: Image Generation Agent（60%）
  ├─ Manus API 生成 Hero 圖片
  ├─ Unsplash API 生成亮點圖片
  ├─ 上傳至 S3
  └─ 風格一致性驗證
  
Step 6: 最終資料組裝（95%）
  ├─ 整合所有 Agent 的結果
  └─ 準備儲存至資料庫
  
Step 7: 儲存至資料庫（100%）
  ├─ 建立 tour 記錄
  ├─ 標記 isAutoGenerated = 1
  ├─ 儲存 sourceUrl 和 originalityScore
  └─ 返回 tourId
```

## 🤖 AI Agent Prompt 架構

### 1. Web Scraper Agent
**System Prompt:**
```
你是一位專業的旅遊資料分析師，擅長從網頁中提取結構化的旅遊資訊。
```

**User Prompt:**
```
請從以下網頁內容中提取旅遊行程資訊，並以 JSON 格式返回：

網頁內容：
{pageContent}

請提取以下資訊：
1. 基本資訊（標題、描述、天數、價格）
2. 目的地資訊（國家、城市）
3. 住宿資訊（飯店名稱、星級、描述）
4. 景點資訊（名稱、描述）
5. 亮點資訊（3-5 個）
6. 航班資訊（航空公司、時間）
7. 餐食資訊（早餐、中餐、晚餐）

返回格式：
{
  "basicInfo": {...},
  "location": {...},
  "accommodation": {...},
  "attractions": [...],
  "highlights": [...],
  "flights": {...},
  "meals": {...}
}
```

### 2. Content Analyzer Agent

#### 2.1 標題改寫
**System Prompt:**
```
你是一位資深旅遊編輯，擅長撰寫吸引人的行銷標題。
```

**User Prompt:**
```
請根據以下資訊重新撰寫一個簡潔、吸引人的行銷標題（不超過 30 字）：

原標題：{originalTitle}
目的地：{destination}
天數：{days}天

要求：
1. 不要原文照抄
2. 突出行程亮點和特色
3. 使用吸引人的詞彙
4. 保持專業和優雅的語調
5. 只回傳標題文字，不要其他說明

範例：「北海道雪國秘境 5 日｜米其林溫泉旅宿 × 洞爺湖心中島探索」
```

#### 2.2 描述改寫
**System Prompt:**
```
你是一位資深旅遊編輯，擅長撰寫精彩的行程介紹。
```

**User Prompt:**
```
請根據以下資訊重新撰寫一段精彩的行程介紹（100-150 字）：

原描述：{originalDescription}
目的地：{destination}
行程亮點：{highlights}

要求：
1. 完全重寫，不要原文照抄
2. 突出行程的獨特性和吸引力
3. 使用優雅、詩意的語言
4. 字數控制在 100-150 字
5. 只回傳介紹文字，不要其他說明

範例：「在北海道的雪白世界中，展開一場心靈的洗禮之旅。入住米其林一星鑰旅宿，感受極致奢華與日式美學的完美融合；搭乘遊船探索洞爺湖心中島，欣賞湖光山色的絕美景致；漫步二世谷秘境，體驗北國獨有的寧靜與感動。這不僅是一趟旅行，更是一場與自然、與自我的深度對話。」
```

#### 2.3 原創性驗證
**System Prompt:**
```
你是一位內容審查專家，擅長評估文案的原創性。
```

**User Prompt:**
```
請評估以下文案的原創性（0-100 分）：

標題：{title}
描述：{description}
Hero 副標題：{heroSubtitle}

評分標準：
1. 60-69 分：基本改寫，仍有明顯的原文痕跡
2. 70-79 分：中度改寫，部分內容有原創性
3. 80-89 分：高度改寫，大部分內容原創
4. 90-100 分：完全原創，無原文痕跡

只回傳分數（整數），不要其他說明。
```

### 3. Image Prompt Agent
**System Prompt:**
```
你是一位專業的 AI 圖片生成提示詞工程師，擅長撰寫高品質的圖片生成提示詞。
```

**User Prompt:**
```
請根據以下資訊生成一個高品質的圖片生成提示詞（100-150 字）：

目的地：{destination}
場景類型：{sceneType} (hero/highlight/feature)
場景描述：{sceneDescription}
風格指南：{styleGuide}

要求：
1. 使用英文撰寫
2. 包含具體的視覺元素（顏色、光線、構圖）
3. 符合風格指南的要求
4. 避免出現文字、商標、人物臉部特寫
5. 只回傳提示詞，不要其他說明

範例：「A breathtaking aerial view of Hokkaido's snow-covered landscape during winter, featuring pristine white mountains, frozen lakes reflecting the golden sunset, and traditional Japanese hot spring resorts nestled in the valley. Soft pastel colors, cinematic lighting, ultra-high resolution, professional travel photography style.」
```

### 4. Color Theme Agent
**System Prompt:**
```
你是一位專業的色彩設計師，擅長根據目的地特色生成配色主題。
```

```javascript
  "default": { primary: "#2980B9", secondary: "#ECF0F1", ... }

```

## 📊 資料庫 Schema

### tours 表（已擴充）
```sql
-- 基本資訊
title VARCHAR(255)
description TEXT
duration INT
price DECIMAL(10,2)

-- 目的地資訊
country VARCHAR(100)
city VARCHAR(100)
destinationDescription TEXT

-- Hero 區域
heroImage VARCHAR(512)
heroImageAlt VARCHAR(255)
heroSubtitle TEXT

-- 配色主題
colorTheme TEXT -- JSON: {primary, secondary, accent, text, textLight, background, backgroundDark}

-- 關鍵特色
keyFeatures TEXT -- JSON array: [{id, keyword, keywordStyle, image, imageAlt, phrases, description}]

-- 詩意文案
poeticContent TEXT -- JSON: {intro, accommodation, dining, experience, closing}
poeticSubtitle TEXT -- 詩意副標題

-- 詳細內容區塊（參考 sipincollection.com）
attractions TEXT -- JSON array: [{name, nameEn, poeticSubtitle, description, image, imageAlt, tags}]
hotels TEXT -- JSON array: [{name, stars, description, image, imageAlt}]
meals TEXT -- JSON array: [{breakfast, lunch, dinner}]
flights TEXT -- JSON: {outbound: [...], inbound: [...]}

-- 每日行程
dailyItinerary TEXT -- JSON array: [{day, title, intro, attractions, meals, hotel}]

-- 管理員資訊
sourceUrl VARCHAR(512) -- 原始來源 URL
originalityScore DECIMAL(5,2) -- 原創性評分（0-100）
isAutoGenerated INT -- 是否為 AI 自動生成（0/1）
```
